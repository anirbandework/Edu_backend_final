# EduAssist Backend - Project Overview

## Project Description
EduAssist is a comprehensive educational management system backend built with FastAPI and PostgreSQL. It provides APIs for managing schools, students, teachers, assessments, and educational workflows.

## Technology Stack
- **Backend Framework**: FastAPI (Python)
- **Database**: PostgreSQL 14
- **Cache**: Redis
- **ORM**: SQLAlchemy (async)
- **Authentication**: JWT-based auth with RBAC
- **Containerization**: Docker & Docker Compose
- **Server**: Gunicorn with Uvicorn workers

## Architecture
**Modular Management-Based Architecture**: Each domain has its own module with models, services, routers, and schemas.

### Core Modules:
1. **auth_rbac/** - Authentication, authorization, roles, permissions
2. **tenant_management/** - Multi-tenant support
3. **student_management/** - Student profiles, enrollment
4. **teacher_management/** - Teacher profiles, assignments
5. **class_management/** - Class creation, management
6. **school_authority_management/** - School admin functions
7. **assessment_management/** - Quizzes, AI-powered assessments, grading
8. **exam_management/** - Formal exams, results
9. **attendance_management/** - Attendance tracking, reports
10. **timetable_management/** - Schedule management
11. **notification_management/** - System notifications
12. **chat_management/** - Real-time messaging
13. **enrollment_management/** - Student-class enrollment

### Each Module Structure:
```
module_name/
├── models/          # SQLAlchemy models
├── routers/         # FastAPI route handlers
├── schemas/         # Pydantic schemas
├── services/        # Business logic
└── __init__.py
```

## Key Features

### Assessment System (Primary Focus)
- **AI-Powered Quiz Generation**: Automatic question generation using AI
- **CBSE Curriculum Support**: Aligned with Indian education standards
- **Multiple Question Types**: MCQ, True/False, Short Answer, Essay
- **Auto-Grading**: Intelligent grading with AI for subjective answers
- **Performance Analytics**: Student progress tracking and insights
- **Adaptive Learning**: Personalized recommendations

### Multi-Tenant Architecture
- Tenant isolation at database level
- Role-based access control (RBAC)
- Hierarchical permissions system

### Real-Time Features
- WebSocket-based chat system
- Live notifications
- Real-time attendance updates

## Database Schema
- **Base Model**: All models inherit from Base with common fields (id, created_at, updated_at, tenant_id)
- **Relationships**: Proper foreign key relationships between entities
- **Indexes**: Optimized for performance with strategic indexing
- **Migrations**: Alembic for database version control

## API Structure
- **RESTful APIs**: Standard HTTP methods and status codes
- **Async Operations**: Full async/await support
- **Request/Response Models**: Pydantic validation
- **Error Handling**: Comprehensive error responses
- **Documentation**: Auto-generated OpenAPI/Swagger docs

## Security
- **JWT Authentication**: Secure token-based auth
- **Password Hashing**: Bcrypt for password security
- **RBAC**: Role-based access control
- **Tenant Isolation**: Data separation between tenants
- **Input Validation**: Pydantic schema validation

## Performance Optimizations
- **Connection Pooling**: SQLAlchemy async engine pools
- **Redis Caching**: Frequently accessed data caching
